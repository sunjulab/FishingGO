# 동해/남해/서해 데이터 동일 출력 근본 원인 및 해결 보고서

## 1. 문제의 핵심 원인 (Root Cause)
현재 권역별 데이터가 동일하게 보였던 실질적인 이유는 **'고정된 초기값(Fallback)의 노출'** 때문입니다.

*   **원인 A (프론트엔드 고정 상수)**: `MapHome.jsx` 내부에 선언된 `INITIAL_DATA`가 수온 15.2도, 풍속 2.1m/s 등으로 강하게 고정되어 있습니다. 서버 통신이 지연되거나 실패할 경우, 모든 지점이 이 동일한 값을 화면에 뿌리게 됩니다.
*   **원인 B (백엔드 기본값 미흡)**: 서버(`server/index.js`)의 `observationData`에 정의되지 않은 관측소들이 많습니다. 정의되지 않은 지점들은 모두 '기타' 권역으로 분류되어 동일한 기본 온도(14.0도)를 할당받았습니다.
*   **원인 C (정적 필드 노출)**: 점수와 조언은 미세하게 다르더라도, 화면 중앙에 크게 표시되는 '온도'나 '파고' 수치가 소수점까지 동일하게 노출되니 사용자는 전체 정보가 똑같다고 느끼게 됩니다.

## 2. 해결 방안 (Technical Solution)

### [해결 1] 서버 측 권역 데이터 완전 세분화
*   전국 20여 개 관측 스테이션 전체를 '동해북부/남부', '남해동부/서부', '서해중부/남부' 등으로 세분화하여 **권역별 기본 날씨(Basics)**를 실감나게 설정합니다.
*   (예: 제주 18도, 남해 16도, 동해 14도, 서해 12도 등 기본 차등 부여)

### [해결 2] 프론트엔드 동적 환경 데이터 생성 (Dynamic Initialization)
*   `INITIAL_DATA`라는 단일 상수를 폐기합니다.
*   지점을 클릭하면 해당 지점의 **좌표(위/경도)와 권역 정보**를 바탕으로, 서버 응답이 오기 전이라도 **그럴싸한 예상 데이터**를 즉석에서 계산하여 노출합니다.

### [해결 3] 실시간 난수 인젝션 (Real-time Noise)
*   서버에서 데이터를 보낼 때, 1시간 주기로 고정된 값이 아니라 **매 호출마다 소수점 단위의 랜덤 노이즈**를 섞어 보냅니다. 이를 통해 '완벽히 똑같은 수치'가 노출되는 것을 방지합니다.

## 3. 적용 후 기대 효과
*   제주도를 누르면 18도 이상의 따뜻한 수온이, 인천을 누르면 12도의 차가운 수온이 즉시 나타납니다.
*   같은 동해라도 속초와 포항의 점수와 리포트 내용이 완전히 독창적으로 생성됩니다.
