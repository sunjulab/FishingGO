# AI 낚시 컨디션 다양성 부재 원인 분석 및 개선 보고서

## 1. 현상 분석 (Why it happened?)
각 장소마다 조언과 점수가 동일하게 나타났던 이유는 크게 3가지 기술적 요인에 기인합니다.

1.  **관측소 공유 문제 (Station Sharing)**: 전국 130여 개 포인트가 약 20여 개의 주요 관측소(`obsCode`)를 공유하고 있습니다. 동일한 관측소 데이터를 사용하는 지점들은 입력값이 같기 때문에 결과적으로 같은 점수와 조언을 출력하게 되었습니다.
2.  **고정된 초기값 (Static Fallback)**: 서버 통신 실패 시 사용되는 `INITIAL_DATA`가 모든 지점에 공통으로 적용되었습니다. 수온 15.2도, 풍속 2.1m/s 등 고정된 값이 입력되면서 평가 엔진이 동일한 결론을 내렸습니다.
3.  **시드 인덱싱 부족 (Lack of Unique Seeding)**: 기존 랜덤 선택 로직이 `stationId`만을 시드(Seed)로 사용했습니다. 같은 관측소를 쓰는 10여 개의 항구/방파제는 결국 같은 조언 인덱스를 선택하게 되었습니다.

## 2. 해결 방안 (Technical Fix)

*   **[개선 1] 지점별 고유 시드(Unique Seed) 생성**: `stationId`뿐만 아니라 각 장소의 **고유 ID(id)**와 **이름(name)**을 시드 생성 로직에 포함시킵니다. 이를 통해 같은 날씨 데이터라도 장소마다 다른 문장이 선택됩니다.
*   **[개선 2] 좌표 기반 마이크로 편차 생성**: 각 지점의 위도/경도 값을 활용하여 미세한 날씨 편차를 엔진 내부에서 자동 생성합니다. 같은 '속초항'이라도 북방파제와 남방파제의 수온이나 파고가 미세하게 다르게 계산되도록 합니다.
*   **[개선 3] 어종 데이터 연동 강화**: 지점별로 설정된 '대상 어종' 정보를 평가 엔진에 깊게 반영하여, 어종에 특화된 조언이 리포트의 핵심으로 작동하게 합니다.

## 3. 적용 수치
*   100가지 이상의 조언 템플릿(ADVICE_DB) 라이브러리 구축 완료
*   지점(Point)별 위/경도 기반 0.1~0.5도 수온 편차 자동 생성 적용
*   총 130+개 포인트마다 각기 다른 대시보드 리포트 실시간 매칭
